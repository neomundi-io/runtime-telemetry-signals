# Runtime Telemetry Notes

This document contains public methodological notes regarding NeoMundi runtime telemetry concepts.

## Current Public Signals

### G

Governance-oriented normalized stability state associated with a generation or execution state.

G is intended to represent a governance-oriented behavioral stability condition.

It is not intended to represent:
- semantic truth,
- factual certainty,
- exhaustive cognition reconstruction,
- or universal model understanding.

---

### ΔG

Runtime variation signal associated with temporal behavioral variation across execution windows.

ΔG is intended for:
- runtime observability,
- instability surfacing,
- behavioral variation monitoring,
- and governance telemetry workflows.

ΔG does not claim:
- semantic omniscience,
- universal hallucination detection,
- or exhaustive runtime reconstruction.

---

## Methodological Position

NeoMundi runtime telemetry signals are governance-oriented observability artifacts.

They are intended to support:
- runtime supervision,
- governance workflows,
- observability systems,
- auditability,
- and orchestration environments.

The repository intentionally separates:
- public conceptual definitions,
- governance observability concepts,
- and proprietary implementation mechanisms.
